

요청하신 대로 app.py, core_engine.py, database.py 3개의 핵심 파일 곳곳에 💡 [AI 스터디 포인트]라는 꼬리표와 함께 각 로직이 어떤 AI 기술(RAG, LangGraph 등)을 활용하고 있는지 상세한 한국어 주석을 달아두었습니다! 에디터에서 해당 파일들을 열어보시면 생생한 스터디 교재로 활용하실 수 있습니다.

🧠 [전체 흐름 브리핑: AI Architecture]
현재 구축된 '개인화 영어 튜터 시스템'은 최신 AI 에이전트 설계 패턴인 "LangGraph 기반의 RAG + Self-RAG 복합 파이프라인" 구조를 띠고 있습니다.

1. 문제 출제 (Zero-Shot Generation)

위치: generate_question (core_engine.py)
유저의 레벨, 주제, 목표 문법을 프롬프트에 동적으로 조립하여 LLM에게 전달합니다. 데이터베이스 없이 오직 프롬프트 튜닝만으로 수준에 맞는 문제를 당장 창작(Generation)하게 만드는 초기 단계입니다. (창의성을 위해 온도(Temperature)를 조절하는 기법이 들어갔습니다.)
2. 의도 파악 및 라우팅 (Routing)

위치: retrieve_node (core_engine.py)
유저가 문장을 입력하면 메인 채점 LLM으로 넘기기 전에, 가벼운 판별을 거칩니다. 만약 완벽한 정답이거나 딴소리(질문 등)라면 뒤의 무거운 DB 검색 과정을 아예 패스(Skip) 시켜 비용과 시간을 아끼는 스마트 라우터 역할을 합니다.
3. 기억력 기반 검색 (Vector Retrieval)

위치: database.py (search_history)
유저가 틀렸을 경우 작동합니다. 유저의 문장을 숫자의 배열(Embedding Vector)로 쪼개어, 과거 오답 DB 코사인 유사도 연산을 통해 "가장 의미가 비슷한 오답"을 찾아옵니다.
특히 이때 문법 태그가 일치하면 강제로 점수를 더 주는 하이브리드 재정렬(Re-ranking) 기법을 사용하여 검색 정확도를 극대화했습니다.
4. 개인화 피드백 생성 (RAG - Retrieval-Augmented Generation)

위치: feedback_node (core_engine.py)
위 3번에서 찾아온 과거의 오답 기록들을 프롬프트에 몰래 끼워 넣은(주입한) 후 채점관 LLM에게 넘깁니다. LLM은 원래 유저가 과거에 뭘 틀렸는지 모르지만, 이 동적으로 주입된 텍스트 덕분에 "지난번에도 관사를 빼먹었는데 또 틀리셨네요!" 와 같이 소름 돋게 기억력 좋은 맞춤형 피드백을 내놓게 됩니다.
5. 자가 검증 (Self-RAG)

위치: verify_node (core_engine.py)
생성형 AI 특유의 환각(헛소리)을 막기 위한 최후의 안전장치입니다. 튜터 LLM이 내놓은 채점 결과가 정말 출제 의도에 맞는지, 문법 설명이 억지가 아닌지를 온도(Temperature)가 0인 매우 차갑고 냉정한 '수석 교사 LLM'이 한 번 더 검증하고, 이상하면 단호하게 덮어써버립니다.
6. State 관리와 그래프 조립 (LangGraph)

위치: build_tutor_graph (core_engine.py)
위 2~5번의 복잡한 과정들을 거치면 변수 관리가 꼬이기 쉽습니다. 이를 방지하기 위해 TutorState라는 단일 딕셔너리 객체를 각 노드(함수)들이 릴레이 바통처럼 전달받으면서 데이터를 채워나가는 깔끔한 상태 머신(State Machine) 구조로 묶어두었습니다.
단순한 챗봇을 넘어서서 아주 훌륭한 LLM Agent Pipeline이 완성되어 가고 있습니다! 추가로 설명해 드렸으면 하는 부분이나 더 다듬고 싶은 코드 로직이 있다면 편하게 말씀해 주세요.
