## 아키텍처 요약

**LangGraph 기반 RAG + Self-RAG 복합 파이프라인**

```
유저 영작 제출
    ↓
[retrieve_node]  — 의도 분류(translation/question/unrelated/new_question)
                 — 오답 시: pgvector 하이브리드 검색 (코사인 유사도 + 문법태그 +0.5 부스팅)
    ↓
[feedback_node]  — history_context 주입(RAG) → 개인화 피드백 생성
    ↓
[verify_node]    — Self-RAG: 피드백 품질 검증, 실패 시 feedback_node 재실행 (최대 2회)
    ↓
[save_node]      — 오답만 벡터화 후 DB 저장 (정답은 저장 안 함)
```

## 모델 구성 (현재 확정)

| 위치 | 모델 | temperature | 비고 |
|---|---|---|---|
| 전역 `llm` (의도분류, 피드백) | `gpt-4o-mini` | 0.5 | 비용 절감 |
| `generate_question` (문제 출제) | `gpt-4o` | 0.8 | 다양성 확보 |
| `generate_review_question` (복습 출제) | `gpt-4o` | 0.7 | |
| `verify_node` (Self-RAG 검증) | `gpt-4o` | 0.0 | 엄격한 판단 필요 |

> `gpt-4o-mini`로 전체 전환 테스트 결과 문제 품질 체감 하락 확인 → 출제/검증은 `gpt-4o` 유지로 결정.

## 구현 완료 기능

### 핵심 파이프라인 (core_engine.py)
- `TutorState` 기반 LangGraph 상태머신
- `retrieve_node`: Intent 분류 + Score 기반 정답 판별(임계값 7점) + 하이브리드 RAG 검색
- `feedback_node`: history_context 프롬프트 주입, Self-RAG 재시도 지원
- `verify_node`: PASS/FAIL 판별, 최대 2회 재생성 루프
- `save_node`: 오답만 선별 저장

### 문제 출제 (core_engine.py)
- `generate_question`: 레벨/주제/타겟문법 기반 Zero-shot 출제
  - `current_question` 파라미터로 직전 문제 전달 → 중복 회피 프롬프팅 적용
  - temperature 0.8로 다양성 확보 (0.3에서 상향)
- `generate_review_question`: 과거 오답 기록 기반 복습 문제 생성

### DB / 검색 (database.py)
- `save_history`: 오답 벡터화(text-embedding-3-small) 후 pgvector 저장
- `search_history`: 코사인 유사도 + 문법태그 Re-ranking 하이브리드 검색
- `get_recent_mistakes`: 최신순 오답 조회 (복습용)
- `get_top_mistake_grammars`: 문법 태그별 오답 횟수 집계 → 많이 틀린 순 Top N 반환 (신규)

### API 서버 (api.py)
- `/api/init`: 세션 초기화 + 첫 문제 생성
- `/api/chat`: 영작 제출 → LangGraph 실행 → 피드백 반환
- `/api/command`: 레벨/주제/문법/모드 변경
  - `!모드 복습`: `get_top_mistake_grammars` 기반 Top 10 복습 모드 진입 (신규)
- `LAST_QUESTION_TEXT`: 직전 문제 저장 후 다음 출제 시 전달

### 프론트엔드 (client/app/page.tsx)
- 레트로 터미널 UI (녹색 테마)
- SETUP_TOPIC → SETUP_GRAMMAR → CHAT 3단계 온보딩
- `!모드 복습` 커맨드 응답 시 복습 문제 화면 출력 (신규)
- 클릭 가능한 주제/문법 선택지

## 잔존 이슈 / 개선 가능 포인트

| 항목 | 내용 |
|---|---|
| 중복 회피 | 직전 1문제만 전달 중 → 최근 N개 누적 전달로 확장 가능 |
| RAG 활용 범위 | Top 1개만 LLM에 전달 중 → Top 3으로 확장 시 컨텍스트 풍부해짐 |
| 복습 모드 트리거 | 수동(`!모드 복습`)만 가능 → 자동 트리거 조건 추가 가능 |
| LangSmith 미연동 | 노드별 입출력 트레이싱 불가 → 디버깅/최적화 시 연동 고려 |
| 세션 관리 | 전역 딕셔너리(단일 유저) → 멀티유저 시 Redis/DB 필요 |
